# Screenshot Guide

Screenshots are the only major manual asset still worth adding after publishing.

The repo is already public-safe and documented, but screenshots will make the GitHub README and proposals feel more visual.

## Recommended capture list

Capture these from the deployed Render URL or local app:

1. Homepage desktop
2. Homepage mobile
3. Starter Kit page desktop
4. Showcase listing page desktop
5. Oakline Decorating showcase detail page
7. Contact form page

## Suggested filenames

If screenshots are added for a public release, place PNGs in a local or tracked `screenshots/` folder using clear names such as:

```text
homepage-desktop.png
homepage-mobile.png
starter-kit-desktop.png
showcase-desktop.png
showcase-oakline-detail.png
case-studies-desktop.png
contact-form.png
```

## README usage

Once screenshots exist, add them near the top of `README.md`, for example:

```md
## Preview

Suggested screenshot names:

```text
homepage-desktop.png
showcase-desktop.png
mobile-home.png
mobile-contact.png
```
```

## Capture tips

- Use Chrome or Edge.
- Capture at normal zoom level.
- Prefer clean sections over long full-page screenshots.
- Avoid browser bookmarks, personal tabs, or local file paths in screenshots.
- Mobile screenshots should show the header/navigation behaving correctly.

## Public safety check

Before committing screenshots, confirm they do not show:

- real private client data
- admin screens
- form submissions
- credentials
- private email addresses
- local machine paths
