# Public Portfolio Notes

This repository is designed to be safe for public GitHub use while still showing real development thinking.

## What is public here

- reusable Express/EJS structure
- industry examples
- placeholder/demo content
- public-safe ASR branding
- sample contact and inquiry flow
- public-safe documentation
- testable project setup

## What stays private

- real client repositories
- production databases
- real form submissions
- private admin systems
- environment secrets
- API keys
- client-specific credentials
- private documents or contracts
- unpublished client branding unless fictionalised or approved

## Good future additions

- screenshots in the README
- a `/showcase-mode` section for fictionalised profile demos
- fictionalised versions of proposal mockups
- short writeups explaining the business problem each fictional demo solves
- a small architecture diagram

## Screenshot guidance

For GitHub presentation, use 3–5 screenshots only:

1. Desktop homepage
2. Mobile homepage/header
3. Services or portfolio grid
4. Contact form
5. Starter kit/profile-switching explanation page

Keep screenshots focused and clean. The goal is to show structure, not every page.
