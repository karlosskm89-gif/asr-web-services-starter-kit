# Public Publishing Checklist

Use this before pushing major changes to the public repo.

## Safety

- [ ] No `.env` file committed
- [ ] `.env.example` contains placeholders only
- [ ] No API keys, tokens, or passwords
- [ ] No private client code
- [ ] No real form submissions or databases
- [ ] No private admin screenshots
- [ ] No real proposal/client branding unless explicitly public and approved

## Repo quality

- [ ] `npm test` passes
- [ ] Main routes return `200`
- [ ] README reflects current routes/features
- [ ] Docs match the actual structure
- [ ] New showcase items are fictionalised
- [ ] New assets use safe placeholder names/contact details

## Presentation

- [ ] GitHub description is clear
- [ ] Topics/tags are set
- [ ] Repo is pinned on profile
- [ ] Live demo link works if included
- [ ] Screenshots are current if referenced
